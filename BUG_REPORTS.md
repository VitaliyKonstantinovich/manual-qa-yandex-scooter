# Баг-репорты

Баг-репорты оформлены по результатам проверки веб- и мобильной части сервиса.

## Баги вне тестовой документации

| ID | Заголовок |
| --- | --- |
| БР32 | В Google Chrome после нажатия кнопки «Заказать» на экране «Про аренду» отображается лишний попап «Хотите оформить заказ?» |
| БР33 | В Google Chrome после подтверждения оформления в попапе «Хотите оформить заказ?» заказ не оформляется |
| БР34 | В окне «Заказ отменён» отображается неверный текст сообщения |
| БР35 | При разрешении 1280x720 окно «Заказ отменён» выходит за границы экрана |
| БР36 | При отмене заказа запрос PUT /api/v1/orders/cancel возвращает 400 Bad Request, но в интерфейсе отображается окно успешной отмены |
| БР27 | Поле «Когда привезти самокат» принимает ввод даты с клавиатуры |
| БР21 | После ввода значения в поле «Станция метро» и нажатия Enter страница очищается |

## Подробные баг-репорты

### БР1. После нажатия на кнопку «Статус заказа» в верхней панели отображается кнопка «GO»

- **Приоритет:** Стандартный
- **Статус:** Открыт
- **Предусловия:** 1. Открыт тестовый стенд.
- **Шаги воспроизведения:** 1. На любой странице нажать кнопку «Статус заказа».
- **Ожидаемый результат:** Вместо кнопок «Заказать» и «Статус заказа» отображается только поле ввода номера заказа.
- **Фактический результат:** Отображается поле ввода номера заказа и дополнительная кнопка «GO».
- **Окружение:** Google Chrome v.147.0.7727.102, Windows 11, 1280x720<br>Google Chrome v.147.0.7727.102, Windows 11, 1920x1080<br>Яндекс.Браузер v.26.3.3.884, Windows 11, 1280x720<br>Яндекс.Браузер v.26.3.3.884, Windows 11, 1920x1080
- **Вложение:** Скриншот

### БР2. После ввода номера заказа в поле верхней панели и нажатия Enter поиск не выполняется

- **Приоритет:** Блокирующий
- **Статус:** Открыт
- **Предусловия:** 1. Открыт тестовый стенд. 2. Создан заказ, получен его номер.
- **Шаги воспроизведения:** 1. Нажать кнопку «Статус заказа» в верхней панели. 2. Ввести номер созданного заказа. 3. Нажать Enter.
- **Ожидаемый результат:** Выполняется поиск по введённому номеру заказа: для существующего заказа открывается страница заказа, для несуществующего отображается сообщение «Такого заказа нет. Точно верный номер?».
- **Фактический результат:** После нажатия Enter поиск по номеру заказа не выполняется.
- **Окружение:** Google Chrome v.147.0.7727.102, Windows 11, 1280x720<br>Google Chrome v.147.0.7727.102, Windows 11, 1920x1080<br>Яндекс.Браузер v.26.3.3.884, Windows 11, 1280x720<br>Яндекс.Браузер v.26.3.3.884, Windows 11, 1920x1080

### БР9. После завершения этапа статус «Ну всё, теперь кататься» не активируется из-за дублирования заказа в БД

- **Приоритет:** Блокирующий
- **Статус:** Открыт
- **Предусловия:** 1. Создан заказ. 2. Через API курьер принял заказ. 3. Через API выполнено завершение заказа.

- **Фактический результат:** В БД создаётся дубль заказа, из-за чего статус не активируется до завершения обоих объектов заказа.
